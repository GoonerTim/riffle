## Описание

`Riffle` (CLI-бинарь и namespace — `riffle`) — потоковый конвертер больших логов из формата **JSON-lines** (по одному
JSON-объекту на строку) в **колоночные форматы**: Apache **Parquet** и сырой columnar-дамп.
Инструмент закрывает нишу между «погрепать лог глазами» и «поднять Spark/pandas/DuckDB ради
одной конвертации»: он превращает терабайтные `.jsonl`-файлы в аналитический Parquet
**на одном ноутбуке, в постоянной памяти и на скорости диска**, не загружая весь вход в RAM.

Модель работы: один или несколько входных потоков (файл, glob, `stdin`) →
потоковый пайплайн (SIMD-парсинг → инференс схемы → колоночная сборка батчами →
запись row-group'ами) → выходной Parquet-файл и статистика прогона.

Поставляется как публичная C++-библиотека (`riffle.hpp`, единая точка входа —
свободная функция `convert`) и как тонкая CLI-обёртка `riffle` поверх неё.

> **Стиль кода.** Functional core + imperative shell: «холодная» логика (конфиг, схема,
> инференс, разрешение конфликтов типов) — чистые свободные функции над неизменяемыми
> value-структурами с фабриками `make_*`; «горячий» путь (буферы чтения и аккумуляторы
> колонок) — изолированный переиспользуемый мутабельный буфер. При конфликте стиля и
> производительности приоритет за производительностью.

## Быстрый старт

### Установка

```bash
# 1. Системные пакеты (Debian/Ubuntu)
apt-get update && apt-get install -y \
    build-essential cmake git \
    libarrow-dev libparquet-dev \
    libzstd-dev libsnappy-dev

# 2. simdjson — header-only, тянется как submodule/FetchContent
git clone https://github.com/simdjson/simdjson third_party/simdjson

# 3. Сборка
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j

# 4. Результат: статический бинарь build/riffle и библиотека build/libriffle.a
ls -lh build/riffle
```

Зависимости: **Apache Arrow** + **Parquet** (`libarrow`, `libparquet`) для записи
колоночного формата; **simdjson** для SIMD-парсинга входа; **zstd**/**snappy** как кодеки
компрессии. Бинарь линкуется статически, рантайм (Python/JVM) не требуется.

### Запуск

```bash
# Файл → Parquet (схема выводится автоматически)
riffle events.jsonl -o events.parquet

# Поток из stdin (вход "-"), сжатие zstd, батч 100k строк
gunzip -c huge.jsonl.gz | riffle - -o out.parquet \
    --compression zstd --batch-rows 100000

# Несколько файлов по glob + явная схема (override инференса)
riffle 'logs/*.jsonl' -o merged.parquet --schema schema.json

# Битые строки складывать в отчёт, не падать; печать статистики в stderr
riffle events.jsonl -o out.parquet --on-error collect --stats
```

### Проверка, что инструмент работает

```bash
# Сконвертировать семпл и прочитать результат сторонним инструментом
printf '%s\n' \
  '{"ts":"2026-06-24T10:00:00Z","level":"info","code":200,"latency_ms":12.5}' \
  '{"ts":"2026-06-24T10:00:01Z","level":"error","code":500,"latency_ms":840.1}' \
  > sample.jsonl

riffle sample.jsonl -o sample.parquet --stats
# stderr: rows_read=2 rows_written=2 rows_skipped=0 bytes_in=... elapsed_ms=...

python -c "import pyarrow.parquet as pq; print(pq.read_table('sample.parquet').to_pandas())"
# Колонки ts(timestamp), level(string), code(int64), latency_ms(double), 2 строки
```

## Архитектурные требования и ограничения

Раздел задаёт рамки, относительно которых обосновано каждое поле сущностей ([[#Форматы данных]])
и каждый процесс ([[#Функции]]).

### Требования

| ID    | Требование              | Содержание                                                                                       |
| ----- | ----------------------- | ------------------------------------------------------------------------------------------------ |
| `R1`  | Пропускная способность  | Порядок **≥ 1 ГБ/с** входного JSON на одном современном CPU-ядре (за счёт SIMD-парсинга и батчей) |
| `R2`  | Постоянная память       | Потребление RAM **O(1) от размера входа**: ограничено `MAX_BATCH_BYTES` + буферы, не размером файла |
| `R3`  | Потоковость             | Однопроходная обработка; данные не материализуются целиком — читаются и пишутся скользящим окном |
| `R4`  | Устойчивость к битым данным | Битая/невалидная строка не роняет процесс при `OnError ≠ abort`; учитывается в `ConvertStats`  |
| `R5`  | Корректность типов      | Колонка получает один Arrow-тип; конфликты разрешаются детерминированно по `TypeConflictPolicy`  |
| `R6`  | Наблюдаемость           | Любой прогон даёт измеримый `ConvertStats`; прогресс выводится для долгих конвертаций            |
| `R7`  | Простота развёртывания  | Один статический бинарь без внешнего рантайма; библиотека — один заголовок + статическая линковка |

### Ограничения и Non-goals

| Ограничение / Non-goal                          | Пояснение                                                                                  |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------ |
| Не СУБД и не query-движок                        | Никакого SQL, фильтров, агрегаций, индексов — только конвертация формата                    |
| Не читает Parquet обратно                        | Направление одностороннее: JSON-lines → columnar. Чтение Parquet — задача потребителя       |
| Не распределённая обработка                      | Один процесс, одна машина. Шардинг/кластеризация — вне области                              |
| Ограниченная вложенность                         | Вложенные объекты разворачиваются в плоские колонки по пути (`a.b.c`) до `MAX_FLATTEN_DEPTH`; глубже — сериализуются как `string` |
| Не GUI                                            | Только библиотека и CLI                                                                     |
| Без многопоточности в v1                         | Горячий путь однопоточный (упрощает память и backpressure); параллелизм — отдельная задача  |
| Без сетевого ввода/вывода                         | Источники — локальные файлы и `stdin`; выход — локальный файл                               |

## Константы

| Константа               | Тип               | Значение                          | Где используется                                              |
| ----------------------- | ----------------- | --------------------------------- | ------------------------------------------------------------- |
| `DEFAULT_BATCH_ROWS`    | `size_t`          | `65536`                           | [[#append_row]], [[#build_batch]] — порог формирования батча  |
| `MAX_BATCH_BYTES`       | `size_t`          | `256 * 1024 * 1024` (256 МиБ)     | [[#append_row]] — байтовый потолок батча (→ `R2`)             |
| `INFER_SAMPLE_ROWS`     | `size_t`          | `10000`                           | [[#infer_schema]] — размер сэмпла для инференса схемы          |
| `DEFAULT_COMPRESSION`   | `CompressionCodec`| `ZSTD`                            | [[#Config]], [[#open_writer]] — кодек по умолчанию             |
| `DEFAULT_ROW_GROUP_BYTES`| `size_t`         | `128 * 1024 * 1024` (128 МиБ)     | [[#flush_row_group]] — целевой размер row-group в Parquet      |
| `READ_BUFFER_BYTES`     | `size_t`          | `1 * 1024 * 1024` (1 МиБ)         | [[#open_input]], [[#next_line]] — буфер потокового чтения      |
| `MAX_LINE_BYTES`        | `size_t`          | `64 * 1024 * 1024` (64 МиБ)       | [[#next_line]] — лимит длины одной строки (защита от OOM)      |
| `MAX_FLATTEN_DEPTH`     | `int`             | `4`                               | [[#extract_value]] — предел разворачивания вложенных объектов  |
| `INFER_PROMOTE_INT_DOUBLE`| `bool`          | `true`                            | [[#resolve_type_conflict]] — int+double расширяется до double |
| `NULL_SENTINEL`         | `string`          | `""` (пустая строка)              | [[#Форматы данных]] — маркер «значение отсутствует»            |
| `SERVICE_NAME`          | `string`          | `"riffle"`                 | [[#emit_stats]], CLI — имя инструмента в логах/usage           |
| `EXIT_OK`               | `int`             | `0`                               | [[#Коды выхода]] — успешная конвертация                        |
| `EXIT_USAGE`            | `int`             | `2`                               | [[#Коды выхода]] — ошибка аргументов CLI                       |
| `EXIT_INPUT`            | `int`             | `3`                               | [[#Коды выхода]] — ошибка чтения входа                         |
| `EXIT_DATA`             | `int`             | `4`                               | [[#Коды выхода]] — фатальная ошибка данных при `OnError::abort`|
| `EXIT_OUTPUT`           | `int`             | `5`                               | [[#Коды выхода]] — ошибка записи выходного файла               |

## Типы данных

### ColumnType

Логический тип колонки, в который маппится JSON-значение. Определяет Arrow-тип и кодирование.

| Атрибут       | Значение        | Назначение                                                       |
| ------------- | --------------- | --------------------------------------------------------------- |
| `INT64`       | `"int64"`       | Целые числа; Arrow `int64`                                       |
| `DOUBLE`      | `"double"`      | Числа с плавающей точкой; Arrow `double`                        |
| `BOOL`        | `"bool"`        | `true`/`false`; Arrow `boolean`                                 |
| `STRING`      | `"string"`      | Строки и не разворачиваемые структуры; Arrow `utf8`             |
| `TIMESTAMP`   | `"timestamp"`   | ISO-8601 строка/epoch → Arrow `timestamp[us, UTC]`             |
| `NULLTYPE`    | `"null"`        | Колонка, в сэмпле встреченная только как `null`; Arrow `null`   |

```cpp
enum class ColumnType { INT64, DOUBLE, BOOL, STRING, TIMESTAMP, NULLTYPE };
```

### CompressionCodec

Кодек компрессии страниц Parquet.

| Атрибут  | Значение     | Назначение                                     |
| -------- | ------------ | ---------------------------------------------- |
| `NONE`   | `"none"`     | Без сжатия (максимум скорости записи)          |
| `SNAPPY` | `"snappy"`   | Быстрое сжатие, умеренная степень              |
| `ZSTD`   | `"zstd"`     | Степень/скорость по умолчанию (`DEFAULT_COMPRESSION`) |

```cpp
enum class CompressionCodec { NONE, SNAPPY, ZSTD };
```

### OnError

Политика реакции на невалидную/нераспарсенную строку (→ `R4`).

| Атрибут   | Значение    | Назначение                                                            |
| --------- | ----------- | -------------------------------------------------------------------- |
| `SKIP`    | `"skip"`    | Пропустить строку, увеличить `rows_skipped`, продолжить              |
| `ABORT`   | `"abort"`   | Прервать конвертацию с `EXIT_DATA`                                   |
| `COLLECT` | `"collect"` | Пропустить, но сохранить `ParseError` в отчёт об ошибках             |

```cpp
enum class OnError { SKIP, ABORT, COLLECT };
```

### TypeConflictPolicy

Как разрешается ситуация, когда в одной колонке встречены значения разных типов (→ `R5`).

| Атрибут  | Значение    | Назначение                                                                 |
| -------- | ----------- | ------------------------------------------------------------------------- |
| `WIDEN`  | `"widen"`   | Расширять до общего типа (`int64`+`double`→`double`); иначе → `string`     |
| `STRING` | `"string"`  | Любой конфликт сводить к `string`                                          |
| `ERROR`  | `"error"`   | Конфликт типов — фатальная ошибка схемы                                    |

```cpp
enum class TypeConflictPolicy { WIDEN, STRING, ERROR };
```

### OutputFormat

Формат выходного файла.

| Атрибут        | Значение         | Назначение                                                              |
| -------------- | ---------------- | ---------------------------------------------------------------------- |
| `PARQUET`      | `"parquet"`      | Apache Parquet через Arrow (основной режим)                            |
| `COLUMNAR_RAW` | `"columnar-raw"` | Простой собственный колоночный дамп (отладка/тесты, без зависимости от Parquet-footer) |

```cpp
enum class OutputFormat { PARQUET, COLUMNAR_RAW };
```

### PipelineState

Состояние конечного автомата конвертации. См. [[#PipelineState Finite State Machine]].

| Атрибут       | Значение         | Назначение                                                  |
| ------------- | ---------------- | ---------------------------------------------------------- |
| `INIT`        | `"init"`         | Открыты вход и выход, ничего не прочитано                   |
| `INFER_SCHEMA`| `"infer_schema"` | Идёт сбор сэмпла и инференс схемы                          |
| `CONVERT`     | `"convert"`      | Основной цикл: парсинг строк и заполнение батча            |
| `FLUSH`       | `"flush"`        | Текущий батч/row-group сбрасывается в writer               |
| `FINALIZE`    | `"finalize"`     | Дописывается Parquet-footer, закрывается файл              |
| `DONE`        | `"done"`         | Успешное терминальное состояние                            |
| `ABORTED`     | `"aborted"`      | Прерывание из-за фатальной ошибки (терминальное)           |

```cpp
enum class PipelineState { INIT, INFER_SCHEMA, CONVERT, FLUSH, FINALIZE, DONE, ABORTED };
```

## Форматы данных

> Все структуры — неизменяемые value-типы, собираются фабриками `make_*` (см.
> [[#Фабрики, конструкторы]]). Исключение — `ColumnBuilder` и `RecordBatch`, которые живут
> на горячем пути и мутируются в изоляции (→ `R1`).

### Config

Параметры одного запуска конвертации. Иммутабельна, собирается до старта пайплайна.

| Атрибут             | Тип                  | Опциональный                         | Назначение / Обоснование (→ требование)                                |
| ------------------- | -------------------- | ------------------------------------ | --------------------------------------------------------------------- |
| `inputs`            | `vector<string>`     | Нет                                  | Пути входов или `"-"` для stdin (→ `R3`, многофайловый вход)           |
| `output_path`       | `string`             | Нет                                  | Путь выходного файла (→ `R7`)                                          |
| `output_format`     | `OutputFormat`       | Да, по умолчанию `PARQUET`           | Формат выхода                                                          |
| `schema_override`   | `InferredSchema`     | Да, по умолчанию пустая              | Явная схема, перекрывающая инференс (→ `R5`)                           |
| `compression`       | `CompressionCodec`   | Да, по умолчанию `DEFAULT_COMPRESSION`| Кодек Parquet                                                         |
| `batch_rows`        | `size_t`             | Да, по умолчанию `DEFAULT_BATCH_ROWS`| Порог строк в батче (→ `R2`)                                          |
| `on_error`          | `OnError`            | Да, по умолчанию `SKIP`              | Политика битых строк (→ `R4`)                                         |
| `type_conflict`     | `TypeConflictPolicy` | Да, по умолчанию `WIDEN`             | Политика конфликта типов (→ `R5`)                                     |
| `emit_stats`        | `bool`               | Да, по умолчанию `false`             | Печатать ли `ConvertStats` в stderr (→ `R6`)                          |

**Инварианты:** `inputs` непуст; `output_path` непуст; `batch_rows ≥ 1`; если задан
`schema_override`, его инвариант уникальности имён выполнен.

```cpp
struct Config {
    std::vector<std::string> inputs;
    std::string              output_path;
    OutputFormat             output_format  = OutputFormat::PARQUET;
    InferredSchema           schema_override;          // пустая → инференс
    CompressionCodec         compression    = CompressionCodec::ZSTD;
    std::size_t              batch_rows      = DEFAULT_BATCH_ROWS;
    OnError                  on_error        = OnError::SKIP;
    TypeConflictPolicy       type_conflict   = TypeConflictPolicy::WIDEN;
    bool                     emit_stats      = false;
};
```

### ColumnSchema

Описание одной выходной колонки.

| Атрибут     | Тип          | Опциональный                  | Назначение / Обоснование (→ требование)                          |
| ----------- | ------------ | ----------------------------- | --------------------------------------------------------------- |
| `name`      | `string`     | Нет                           | Имя колонки в Parquet (плоский путь, напр. `req.code`) (→ `R5`)  |
| `type`      | `ColumnType` | Нет                           | Логический тип колонки (→ `R5`)                                 |
| `nullable`  | `bool`       | Да, по умолчанию `true`       | Допускает ли колонка `null` (→ `R4`)                           |
| `json_path` | `string`     | Да, по умолчанию = `name`     | Путь к значению в исходном JSON (для разворачивания вложенности) |

**Инварианты:** `name` непуст; `json_path` непуст; `type` определён до создания
[[#ColumnBuilder]].

```cpp
struct ColumnSchema {
    std::string name;
    ColumnType  type;
    bool        nullable  = true;
    std::string json_path;   // == name, если не вложенный
};
```

### InferredSchema

Полная схема выхода — упорядоченный набор колонок плюс диагностика инференса.

| Атрибут          | Тип                    | Опциональный            | Назначение / Обоснование (→ требование)                       |
| ---------------- | ---------------------- | ----------------------- | ------------------------------------------------------------ |
| `columns`        | `vector<ColumnSchema>` | Нет                     | Колонки в порядке появления (→ `R5`)                         |
| `sampled_rows`   | `size_t`               | Да, по умолчанию `0`    | Сколько строк просмотрено при инференсе (→ `R6`)             |
| `had_conflicts`  | `bool`                 | Да, по умолчанию `false`| Были ли разрешённые конфликты типов (→ `R5`, `R6`)          |

**Инварианты:** имена в `columns` **уникальны**; порядок колонок стабилен (детерминизм
выхода); пустой `columns` допустим только для пустого входа.

```cpp
struct InferredSchema {
    std::vector<ColumnSchema> columns;
    std::size_t               sampled_rows = 0;
    bool                      had_conflicts = false;
};
```

### ColumnBuilder

Горячий аккумулятор значений одной колонки внутри текущего батча. **Мутабельный,
переиспользуемый** — после сброса батча очищается, но не освобождает память (→ `R1`, `R2`).

| Атрибут        | Тип                          | Опциональный | Назначение / Обоснование (→ требование)                       |
| -------------- | ---------------------------- | ------------ | ------------------------------------------------------------ |
| `schema`       | `ColumnSchema`               | Нет          | Какую колонку строит (→ `R5`)                                |
| `arrow_builder`| `arrow::ArrayBuilder*`       | Нет          | Типизированный Arrow-builder под `schema.type` (→ `R1`)       |
| `null_count`   | `size_t`                     | Да, по умолч. `0` | Счётчик `null` в текущем батче (→ `R6`)                  |

**Инварианты:** длина `arrow_builder` равна числу обработанных строк батча (все колонки
выровнены); `arrow_builder` соответствует `schema.type`.

### RecordBatch

Готовый колоночный батч для передачи в writer. Тонкая обёртка над `arrow::RecordBatch`.

| Атрибут   | Тип                          | Опциональный | Назначение / Обоснование (→ требование)                  |
| --------- | ---------------------------- | ------------ | ------------------------------------------------------- |
| `arrow`   | `shared_ptr<arrow::RecordBatch>` | Нет      | Колоночные данные батча (→ `R1`)                        |
| `n_rows`  | `size_t`                     | Нет          | Число строк (≤ `batch_rows`) (→ `R2`)                  |

**Инварианты:** все колонки имеют длину `n_rows`; `n_rows ≤ Config.batch_rows`;
суммарный размер ≤ `MAX_BATCH_BYTES`.

### ParseError

Запись об одной непринятой строке (заполняется при `OnError::COLLECT`).

| Атрибут     | Тип      | Опциональный                | Назначение / Обоснование (→ требование)                       |
| ----------- | -------- | --------------------------- | ------------------------------------------------------------ |
| `line_no`   | `size_t` | Нет                         | Номер строки во входе (1-based) (→ `R4`, `R6`)               |
| `reason`    | `string` | Нет                         | Причина (невалидный JSON / тип / превышен `MAX_LINE_BYTES`)   |
| `raw`       | `string` | Да, по умолчанию `NULL_SENTINEL` | Сырой фрагмент строки (усечённый) для диагностики        |

**Инварианты:** `line_no ≥ 1`; `reason` непуст.

```cpp
struct ParseError {
    std::size_t line_no;
    std::string reason;
    std::string raw;   // усечён до разумного предела
};
```

### ConvertStats

Итог прогона — единственный наблюдаемый результат `convert` (→ `R6`).

| Атрибут        | Тип                  | Опциональный | Назначение / Обоснование (→ требование)                  |
| -------------- | -------------------- | ------------ | ------------------------------------------------------- |
| `rows_read`    | `size_t`             | Нет          | Прочитано строк входа (→ `R6`)                          |
| `rows_written` | `size_t`             | Нет          | Записано строк в выход (→ `R6`)                         |
| `rows_skipped` | `size_t`             | Нет          | Пропущено битых строк (→ `R4`, `R6`)                   |
| `bytes_in`     | `size_t`             | Нет          | Прочитано байт входа (→ `R6`, расчёт ГБ/с)             |
| `bytes_out`    | `size_t`             | Нет          | Размер выходного файла (→ `R6`)                        |
| `elapsed_ms`   | `uint64`             | Нет          | Длительность прогона (→ `R1`, `R6`)                    |
| `final_state`  | `PipelineState`      | Нет          | `DONE` или `ABORTED` (→ `R6`)                          |
| `errors`       | `vector<ParseError>` | Да, по умолч. `[]` | Собранные ошибки при `OnError::COLLECT`             |

**Инварианты:** `rows_written + rows_skipped ≤ rows_read`; `final_state ∈ {DONE, ABORTED}`;
при `ABORTED` поля-счётчики отражают прогресс на момент прерывания.

```cpp
struct ConvertStats {
    std::size_t             rows_read    = 0;
    std::size_t             rows_written = 0;
    std::size_t             rows_skipped = 0;
    std::size_t             bytes_in     = 0;
    std::size_t             bytes_out    = 0;
    std::uint64_t           elapsed_ms   = 0;
    PipelineState           final_state  = PipelineState::INIT;
    std::vector<ParseError> errors;
};
```

## Машина состояний

### PipelineState Finite State Machine

Конвертация моделируется как линейный автомат с двумя терминальными состояниями. Переход
выполняет [[#convert]]; на каждом шаге он обновляет аккумулятор статистики ([[#ConvertStats]]).

```
INIT ──open ok──▶ INFER_SCHEMA ──schema ready──▶ CONVERT ──┐
                                                    ▲       │ батч полон / EOF
                                                    │       ▼
                                                    └────── FLUSH ──more input──▶ CONVERT
                                                              │ EOF
                                                              ▼
                                                          FINALIZE ──footer ok──▶ DONE

любое состояние ──фатальная ошибка / OnError::ABORT──▶ ABORTED
```

| Переход                  | Условие                                                      | Функция                          |
| ------------------------ | ----------------------------------------------------------- | -------------------------------- |
| `INIT → INFER_SCHEMA`     | Вход и выход открыты                                        | [[#open_input]], [[#open_writer]] |
| `INFER_SCHEMA → CONVERT`  | Собран сэмпл (`INFER_SAMPLE_ROWS` либо EOF) и выведена схема | [[#infer_schema]], [[#merge_override]] |
| `CONVERT → FLUSH`         | Батч достиг `batch_rows`/`MAX_BATCH_BYTES` либо EOF         | [[#build_batch]]                 |
| `FLUSH → CONVERT`         | После сброса есть ещё вход                                  | [[#write_batch]]                 |
| `FLUSH → FINALIZE`        | После сброса достигнут EOF всех входов                      | [[#flush_row_group]]             |
| `FINALIZE → DONE`         | Footer/закрытие файла успешны                               | [[#finalize_file]]               |
| `* → ABORTED`             | Фатальная ошибка I/O/схемы или `OnError::ABORT` на строке   | [[#handle_parse_error]]          |

**Инварианты FSM:** из `DONE`/`ABORTED` переходов нет; в `FINALIZE` входят, только когда
все входы прочитаны; счётчики `ConvertStats` монотонно не убывают.

## Функции

> Каждая функция описана как **What** (поведение, edge-кейсы, ограничения) и **Why** (какому
> требованию/инварианту служит). Сигнатуры — свободные функции; состояние горячего пути
> передаётся явно через `PipelineCtx` (структура-аккумулятор: открытые потоки, набор
> [[#ColumnBuilder]], writer, накопленный [[#ConvertStats]]).

### Чтение

#### open_input

**Описание.** Открывает следующий вход (файл или `stdin`) для потокового чтения.

**Сигнатура:** `auto open_input(const std::string& path) -> InputStream`

**Входные данные:**

| Атрибут | Тип      | Опциональный | Назначение                                  |
| ------- | -------- | ------------ | ------------------------------------------- |
| `path`  | `string` | Нет          | Путь к файлу или `"-"` для `stdin`          |

**Выходные данные:**

| Атрибут  | Тип           | Опциональный | Назначение                                         |
| -------- | ------------- | ------------ | -------------------------------------------------- |
| `return` | `InputStream` | Нет          | Поток с внутренним буфером `READ_BUFFER_BYTES`     |

**Side effects:** открывает файловый дескриптор / захватывает `stdin`.

**Логика / What.** Открывает `path`; для `"-"` берёт `stdin`. Выделяет буфер чтения
`READ_BUFFER_BYTES` один раз и переиспользует его. На несуществующий/недоступный файл —
ошибка, ведущая к `EXIT_INPUT`. **Why:** единая точка потокового ввода обеспечивает
постоянную память (→ `R2`) и поддержку pipe-источников (→ `R3`).

#### next_line

**Описание.** Возвращает следующую логическую строку (JSON-объект) без копирования, как
`string_view` в буфер чтения.

**Сигнатура:** `auto next_line(InputStream& in) -> std::optional<std::string_view>`

**Входные данные:**

| Атрибут | Тип            | Опциональный | Назначение                  |
| ------- | -------------- | ------------ | --------------------------- |
| `in`    | `InputStream&` | Нет          | Поток из [[#open_input]]    |

**Выходные данные:**

| Атрибут  | Тип                            | Опциональный | Назначение                                    |
| -------- | ------------------------------ | ------------ | --------------------------------------------- |
| `return` | `optional<string_view>`        | Да           | Строка без `\n`; `nullopt` — EOF              |

**Side effects:** дочитывает данные в буфер по мере необходимости (мутирует `in`).

**Логика / What.** Ищет `\n` в буфере; при необходимости дозагружает `READ_BUFFER_BYTES`.
Возвращает view на участок буфера (zero-copy). Строка, переходящая границу буфера,
дособирается без полного удвоения. Если длина строки превышает `MAX_LINE_BYTES` — не
аллоцирует бесконечно, а возвращает маркер ошибки (битая строка). Пустые строки
пропускаются. **Why:** zero-copy чтение — основа пропускной способности (→ `R1`) и
ограниченной памяти (→ `R2`); лимит `MAX_LINE_BYTES` защищает от OOM на «склеенном» вводе.

### Парсинг

#### parse_line

**Описание.** Парсит одну строку JSON через simdjson (on-demand API).

**Сигнатура:** `auto parse_line(simdjson::ondemand::parser& p, std::string_view line) -> std::expected<simdjson::ondemand::document, ParseError>`

**Входные данные:**

| Атрибут | Тип                       | Опциональный | Назначение                              |
| ------- | ------------------------- | ------------ | --------------------------------------- |
| `p`     | `ondemand::parser&`       | Нет          | Переиспользуемый simdjson-парсер        |
| `line`  | `string_view`             | Нет          | Строка из [[#next_line]]                |

**Выходные данные:**

| Атрибут  | Тип                                    | Опциональный | Назначение                              |
| -------- | -------------------------------------- | ------------ | --------------------------------------- |
| `return` | `expected<document, ParseError>`       | Нет          | Документ или причина отказа             |

**Side effects:** нет (парсер переиспользует свой буфер).

**Логика / What.** Запускает SIMD-парсер на строке. Требует объект верхнего уровня
(`{...}`); массив/скаляр верхнего уровня → `ParseError("not an object")`. Невалидный JSON →
`ParseError` с позицией. Парсер `p` создаётся один раз и переиспользуется между строками.
**Why:** SIMD-парсинг — ключевой источник скорости (→ `R1`); типобезопасный возврат ошибки
без исключений держит горячий путь предсказуемым (→ `R4`).

#### extract_value

**Описание.** Извлекает значение колонки из документа по `json_path`, приводя к
`ColumnType` колонки; разворачивает вложенность до `MAX_FLATTEN_DEPTH`.

**Сигнатура:** `auto extract_value(simdjson::ondemand::document& doc, const ColumnSchema& col) -> CellValue`

**Входные данные:**

| Атрибут | Тип                   | Опциональный | Назначение                          |
| ------- | --------------------- | ------------ | ----------------------------------- |
| `doc`   | `ondemand::document&` | Нет          | Распарсенный документ строки        |
| `col`   | `const ColumnSchema&` | Нет          | Целевая колонка и её `json_path`    |

**Выходные данные:**

| Атрибут  | Тип         | Опциональный | Назначение                                          |
| -------- | ----------- | ------------ | --------------------------------------------------- |
| `return` | `CellValue` | Нет          | Типизированное значение или `null`                  |

**Side effects:** нет.

**Логика / What.** Идёт по `json_path` (сегменты через `.`). Отсутствие ключа / `null` →
`null`-значение (`null_count++` в [[#append_row]]). Значение приводится к `col.type`
(число→int64/double, строка ISO→timestamp и т.д.). Вложенный объект/массив на пути глубже
`MAX_FLATTEN_DEPTH` сериализуется в `string`. Несоответствие типа значения и `col.type`
обрабатывается согласно [[#TypeConflictPolicy]] (на этапе сборки). **Why:** детерминированное
извлечение и плоское представление обеспечивают корректную типизированную колонку (→ `R5`)
и удерживают сложность в рамках Non-goal по вложенности.

### Инференс схемы

#### infer_schema

**Описание.** Выводит [[#InferredSchema]] по сэмплу первых строк входа.

**Сигнатура:** `auto infer_schema(InputStream& in, const Config& cfg) -> InferredSchema`

**Входные данные:**

| Атрибут | Тип            | Опциональный | Назначение                          |
| ------- | -------------- | ------------ | ----------------------------------- |
| `in`    | `InputStream&` | Нет          | Поток входа (будет частично прочитан)|
| `cfg`   | `const Config&`| Нет          | Политики типов и override           |

**Выходные данные:**

| Атрибут  | Тип              | Опциональный | Назначение                          |
| -------- | ---------------- | ------------ | ----------------------------------- |
| `return` | `InferredSchema` | Нет          | Колонки в порядке первого появления |

**Side effects:** читает до `INFER_SAMPLE_ROWS` строк во внутренний буфер сэмпла (память
ограничена, → `R2`); прочитанные строки **не теряются** — кэшируются для последующей конвертации.

**Логика / What.** Берёт до `INFER_SAMPLE_ROWS` строк (или меньше при EOF), парсит каждую,
по каждому встреченному ключу заводит колонку и накапливает множество наблюдённых типов.
Для каждой колонки итоговый тип = свёртка наблюдённых типов через [[#resolve_type_conflict]].
Колонка, всегда `null` в сэмпле, получает `NULLTYPE` и `nullable=true`. Порядок колонок —
порядок первого появления (детерминизм). Сэмпл-строки сохраняются, чтобы конвертация была
однопроходной (→ `R3`). **Why:** инференс даёт zero-config UX, а сэмпл фиксированного размера
сохраняет постоянную память (→ `R2`).

#### merge_override

**Описание.** Накладывает явную схему `Config.schema_override` на выведенную.

**Сигнатура:** `auto merge_override(InferredSchema inferred, const InferredSchema& override) -> InferredSchema`

**Входные данные:**

| Атрибут    | Тип                    | Опциональный | Назначение                           |
| ---------- | ---------------------- | ------------ | ------------------------------------ |
| `inferred` | `InferredSchema`       | Нет          | Результат [[#infer_schema]]          |
| `override` | `const InferredSchema&`| Нет          | Явные колонки от пользователя        |

**Выходные данные:**

| Атрибут  | Тип              | Опциональный | Назначение                  |
| -------- | ---------------- | ------------ | --------------------------- |
| `return` | `InferredSchema` | Нет          | Объединённая схема          |

**Side effects:** нет (чистая функция, принимает `inferred` по значению).

**Логика / What.** Для каждой колонки из `override`: если одноимённая есть в `inferred` —
её `type`/`nullable` заменяются явными; если нет — колонка добавляется. Прочие выведенные
колонки сохраняются. Результат проверяется на инвариант уникальности имён. **Why:**
override закрывает случаи, где инференс ошибается (напр. ID, который выглядит как число, но
должен быть `string`) (→ `R5`), не теряя автоматизма для остальных колонок.

#### resolve_type_conflict

**Описание.** Сводит множество наблюдённых типов колонки к одному по
[[#TypeConflictPolicy]].

**Сигнатура:** `auto resolve_type_conflict(std::span<const ColumnType> seen, TypeConflictPolicy policy) -> std::expected<ColumnType, std::string>`

**Входные данные:**

| Атрибут  | Тип                       | Опциональный | Назначение                          |
| -------- | ------------------------- | ------------ | ----------------------------------- |
| `seen`   | `span<const ColumnType>`  | Нет          | Все типы, встреченные в сэмпле      |
| `policy` | `TypeConflictPolicy`      | Нет          | Политика разрешения                 |

**Выходные данные:**

| Атрибут  | Тип                              | Опциональный | Назначение                          |
| -------- | -------------------------------- | ------------ | ----------------------------------- |
| `return` | `expected<ColumnType, string>`   | Нет          | Итоговый тип либо ошибка схемы       |

**Side effects:** нет.

**Логика / What.** `NULLTYPE` игнорируется при наличии другого типа. Единственный
не-null тип → он и есть результат. При нескольких типах: `WIDEN` — `INT64`+`DOUBLE`→`DOUBLE`
(если `INFER_PROMOTE_INT_DOUBLE`), иначе→`STRING`; `STRING` — всегда `STRING`; `ERROR` —
возвращает ошибку (ведёт к `ABORTED`/`EXIT_DATA`). Только `NULLTYPE` во всём сэмпле →
`NULLTYPE`. **Why:** детерминированное разрешение гарантирует один Arrow-тип на колонку
(инвариант [[#InferredSchema]], → `R5`) и предсказуемый выход независимо от порядка строк.

### Колоночная сборка

#### append_row

**Описание.** Добавляет одну распарсенную строку в текущие [[#ColumnBuilder]] (горячий путь).

**Сигнатура:** `void append_row(PipelineCtx& ctx, simdjson::ondemand::document& doc)`

**Входные данные:**

| Атрибут | Тип                   | Опциональный | Назначение                          |
| ------- | --------------------- | ------------ | ----------------------------------- |
| `ctx`   | `PipelineCtx&`        | Нет          | Контекст с набором builder'ов и схемой |
| `doc`   | `ondemand::document&` | Нет          | Документ строки                     |

**Выходные данные:** нет (мутирует `ctx`).

**Side effects:** дописывает значения во все `arrow_builder`; увеличивает счётчики
строк/`null`.

**Логика / What.** Для каждой колонки схемы вызывает [[#extract_value]] и аппендит результат
в соответствующий `ColumnBuilder` (отсутствие → `null`, `null_count++`). Все колонки
аппендятся ровно раз за строку — этим держится инвариант равной длины. По достижении
`batch_rows` **или** `MAX_BATCH_BYTES` сигнализирует оркестратору о необходимости
[[#build_batch]] (переход `CONVERT→FLUSH`). **Why:** построчный аппенд в переиспользуемые
builder'ы — суть постоянной памяти (→ `R2`) и выровненного батча (инвариант [[#RecordBatch]]).

#### build_batch

**Описание.** Замораживает текущие builder'ы в [[#RecordBatch]] и очищает их под следующий батч.

**Сигнатура:** `auto build_batch(PipelineCtx& ctx) -> RecordBatch`

**Входные данные:**

| Атрибут | Тип            | Опциональный | Назначение                  |
| ------- | -------------- | ------------ | --------------------------- |
| `ctx`   | `PipelineCtx&` | Нет          | Контекст с builder'ами      |

**Выходные данные:**

| Атрибут  | Тип           | Опциональный | Назначение                  |
| -------- | ------------- | ------------ | --------------------------- |
| `return` | `RecordBatch` | Нет          | Готовый колоночный батч     |

**Side effects:** вызывает `Finish()` у Arrow-builder'ов и затем сбрасывает их (`Reset`),
сохраняя выделенную ёмкость.

**Логика / What.** Для каждого `ColumnBuilder` вызывает `Finish()`, собирает
`arrow::RecordBatch`, фиксирует `n_rows`. Затем builder'ы сбрасываются **без освобождения
capacity** — чтобы следующий батч не переаллоцировал. Проверяет инвариант равной длины
колонок. **Why:** заморозка батчами амортизирует запись и держит память постоянной (→ `R2`);
переиспользование ёмкости — для пропускной способности (→ `R1`).

#### widen_column

**Описание.** Расширяет тип уже строящейся колонки, если в `CONVERT`-фазе встретилось
значение шире выведенного типа (только при `TypeConflictPolicy::WIDEN`).

**Сигнатура:** `auto widen_column(ColumnBuilder& col, ColumnType to) -> std::expected<void, std::string>`

**Входные данные:**

| Атрибут | Тип              | Опциональный | Назначение                          |
| ------- | ---------------- | ------------ | ----------------------------------- |
| `col`   | `ColumnBuilder&` | Нет          | Колонка, тип которой расширяется     |
| `to`    | `ColumnType`     | Нет          | Целевой более широкий тип            |

**Выходные данные:**

| Атрибут  | Тип                       | Опциональный | Назначение                          |
| -------- | ------------------------- | ------------ | ----------------------------------- |
| `return` | `expected<void, string>`  | Нет          | Успех или причина невозможности       |

**Side effects:** заменяет `arrow_builder` колонки на builder более широкого типа,
конвертируя уже накопленные значения.

**Логика / What.** Покрывает случай, когда инференс по сэмплу дал `INT64`, а дальше во
входе пришёл `DOUBLE` (сэмпл не покрыл все строки). Допустимые расширения: `INT64→DOUBLE`,
`*→STRING`. Уже накопленные значения текущего батча конвертируются на месте. Недопустимое
расширение → ошибка (→ `ABORTED` при строгой политике). **Why:** делает однопроходную
конвертацию устойчивой к данным за пределами сэмпла (→ `R3`, `R5`), не требуя второго прохода.

### Запись

#### open_writer

**Описание.** Создаёт выходной writer под `OutputFormat` и схему.

**Сигнатура:** `auto open_writer(const Config& cfg, const InferredSchema& schema) -> Writer`

**Входные данные:**

| Атрибут  | Тип                   | Опциональный | Назначение                          |
| -------- | --------------------- | ------------ | ----------------------------------- |
| `cfg`    | `const Config&`       | Нет          | Путь, формат, кодек                  |
| `schema` | `const InferredSchema&`| Нет         | Колонки для Arrow-схемы              |

**Выходные данные:**

| Атрибут  | Тип      | Опциональный | Назначение                          |
| -------- | -------- | ------------ | ----------------------------------- |
| `return` | `Writer` | Нет          | Открытый Parquet/columnar writer    |

**Side effects:** создаёт/обрезает выходной файл.

**Логика / What.** Строит `arrow::Schema` из [[#InferredSchema]]. Для `PARQUET` открывает
`parquet::arrow::FileWriter` с `cfg.compression` и целевым `DEFAULT_ROW_GROUP_BYTES`. Для
`COLUMNAR_RAW` открывает простой собственный writer (заголовок + блоки колонок). Ошибка
создания файла → `EXIT_OUTPUT`. **Why:** единая абстракция writer изолирует формат от
пайплайна (→ простота, тестируемость) и даёт выбор кодека (→ `R7`).

#### write_batch

**Описание.** Пишет один [[#RecordBatch]] в открытый writer.

**Сигнатура:** `auto write_batch(Writer& w, const RecordBatch& batch) -> std::expected<void, std::string>`

**Входные данные:**

| Атрибут | Тип                  | Опциональный | Назначение                  |
| ------- | -------------------- | ------------ | --------------------------- |
| `w`     | `Writer&`            | Нет          | Writer из [[#open_writer]]  |
| `batch` | `const RecordBatch&` | Нет          | Батч из [[#build_batch]]    |

**Выходные данные:**

| Атрибут  | Тип                      | Опциональный | Назначение                  |
| -------- | ------------------------ | ------------ | --------------------------- |
| `return` | `expected<void, string>` | Нет          | Успех или ошибка записи      |

**Side effects:** дописывает данные в выходной файл; увеличивает `bytes_out`.

**Логика / What.** Передаёт `batch.arrow` в Arrow writer. Аккумулирует записанные байты в
текущем row-group. По достижении `DEFAULT_ROW_GROUP_BYTES` инициирует [[#flush_row_group]].
Ошибка I/O возвращается вверх (→ `ABORTED`/`EXIT_OUTPUT`). **Why:** побатчевая запись —
основа потоковости (→ `R3`) и постоянной памяти (→ `R2`).

#### flush_row_group

**Описание.** Закрывает текущий row-group Parquet и начинает новый.

**Сигнатура:** `auto flush_row_group(Writer& w) -> std::expected<void, std::string>`

**Входные данные:**

| Атрибут | Тип       | Опциональный | Назначение                  |
| ------- | --------- | ------------ | --------------------------- |
| `w`     | `Writer&` | Нет          | Активный writer             |

**Выходные данные:**

| Атрибут  | Тип                      | Опциональный | Назначение                  |
| -------- | ------------------------ | ------------ | --------------------------- |
| `return` | `expected<void, string>` | Нет          | Успех или ошибка             |

**Side effects:** материализует страницы row-group на диск.

**Логика / What.** Завершает текущий row-group, сбрасывает буферы кодека на диск, обновляет
метаданные. Для `COLUMNAR_RAW` — no-op либо запись разделителя блока. **Why:** ограниченный
размер row-group (`DEFAULT_ROW_GROUP_BYTES`) держит память записи постоянной (→ `R2`) и
даёт читателю эффективный pruning.

#### finalize_file

**Описание.** Дописывает footer/метаданные и закрывает выходной файл.

**Сигнатура:** `auto finalize_file(Writer& w) -> std::expected<void, std::string>`

**Входные данные:**

| Атрибут | Тип       | Опциональный | Назначение                  |
| ------- | --------- | ------------ | --------------------------- |
| `w`     | `Writer&` | Нет          | Активный writer             |

**Выходные данные:**

| Атрибут  | Тип                      | Опциональный | Назначение                  |
| -------- | ------------------------ | ------------ | --------------------------- |
| `return` | `expected<void, string>` | Нет          | Успех или ошибка             |

**Side effects:** записывает Parquet-footer (schema + статистика row-group'ов), закрывает
дескриптор.

**Логика / What.** Вызывает `Close()` Arrow writer — записывается footer с метаданными
схемы и row-group'ов, без которого файл нечитаем. Для `COLUMNAR_RAW` пишет финальный
индекс блоков. Ошибка на этом этапе фатальна (частичный файл) → `EXIT_OUTPUT`. **Why:**
корректное завершение файла — обязательное условие читаемости Parquet (→ `R5`, корректность).

### Оркестрация

#### convert

**Описание.** Единая точка входа: прогоняет весь пайплайн от входа к выходу и возвращает
статистику. Свободная функция; всё состояние — в локальном `PipelineCtx`.

**Сигнатура:** `auto convert(const Config& cfg) -> ConvertStats`

**Входные данные:**

| Атрибут | Тип            | Опциональный | Назначение                          |
| ------- | -------------- | ------------ | ----------------------------------- |
| `cfg`   | `const Config&`| Нет          | Полная конфигурация прогона          |

**Выходные данные:**

| Атрибут  | Тип            | Опциональный | Назначение                          |
| -------- | -------------- | ------------ | ----------------------------------- |
| `return` | `ConvertStats` | Нет          | Итог прогона (→ `R6`)               |

**Side effects:** читает входы, создаёт и наполняет выходной файл, пишет прогресс/статистику
в stderr при `cfg.emit_stats`.

**Логика / What.** Реализует FSM из [[#PipelineState Finite State Machine]]:
`open_input`+`open_writer` (`INIT`), затем `infer_schema`+`merge_override` (`INFER_SCHEMA`),
далее цикл `CONVERT↔FLUSH` (`next_line`→`parse_line`→`append_row`; по порогу — `build_batch`
+`write_batch`), на EOF — `flush_row_group`+`finalize_file` (`FINALIZE`→`DONE`). Несколько
входов обрабатываются последовательно под общей схемой. Битые строки идут в
[[#handle_parse_error]]. Состояние (builder'ы, writer, `ConvertStats`) живёт в локальном
`PipelineCtx` и передаётся явно — без объекта-оркестратора. Backpressure естественный:
батч пишется и память переиспользуется, прежде чем читать дальше. **Why:** одна свободная
функция как контракт (→ простота, тестируемость), однопроходный цикл с переиспользованием
буферов (→ `R1`, `R2`, `R3`).

#### handle_parse_error

**Описание.** Применяет [[#OnError]] к непринятой строке.

**Сигнатура:** `auto handle_parse_error(PipelineCtx& ctx, const Config& cfg, ParseError err) -> bool`

**Входные данные:**

| Атрибут | Тип            | Опциональный | Назначение                          |
| ------- | -------------- | ------------ | ----------------------------------- |
| `ctx`   | `PipelineCtx&` | Нет          | Контекст (счётчики, отчёт об ошибках)|
| `cfg`   | `const Config&`| Нет          | Политика `on_error`                 |
| `err`   | `ParseError`   | Нет          | Описание ошибки строки              |

**Выходные данные:**

| Атрибут  | Тип    | Опциональный | Назначение                                          |
| -------- | ------ | ------------ | --------------------------------------------------- |
| `return` | `bool` | Нет          | `true` — продолжать; `false` — прервать (`ABORTED`) |

**Side effects:** увеличивает `rows_skipped`; при `COLLECT` добавляет `err` в `ConvertStats.errors`.

**Логика / What.** `SKIP` — `rows_skipped++`, вернуть `true`. `COLLECT` — то же плюс
сохранить `err` (с защитой от неограниченного роста отчёта). `ABORT` — вернуть `false`,
оркестратор переходит в `ABORTED` и завершает с `EXIT_DATA`. **Why:** изолирует политику
ошибок в одном месте (→ `R4`) и сохраняет диагностику без падения процесса (→ `R6`).

### Наблюдаемость

#### emit_stats

**Описание.** Печатает итоговый [[#ConvertStats]] в stderr.

**Сигнатура:** `void emit_stats(const ConvertStats& stats)`

**Входные данные:**

| Атрибут | Тип                  | Опциональный | Назначение          |
| ------- | -------------------- | ------------ | ------------------- |
| `stats` | `const ConvertStats&`| Нет          | Итог прогона        |

**Выходные данные:** нет.

**Side effects:** пишет строку статистики в stderr.

**Логика / What.** Форматирует `rows_read/rows_written/rows_skipped/bytes_in/bytes_out/
elapsed_ms` и производную скорость (ГБ/с) одной строкой с префиксом `SERVICE_NAME`. При
наличии `errors` добавляет их количество. Вызывается из [[#convert]] только если
`Config.emit_stats`. **Why:** даёт измеримый, машиночитаемый итог (→ `R6`) и подтверждает
достижение `R1` на реальных данных.

#### report_progress

**Описание.** Периодически выводит прогресс при долгой конвертации.

**Сигнатура:** `void report_progress(const ConvertStats& running, std::uint64_t every_ms)`

**Входные данные:**

| Атрибут    | Тип                  | Опциональный | Назначение                              |
| ---------- | -------------------- | ------------ | --------------------------------------- |
| `running`  | `const ConvertStats&`| Нет          | Текущие накопленные счётчики            |
| `every_ms` | `uint64`             | Нет          | Минимальный интервал между выводами     |

**Выходные данные:** нет.

**Side effects:** при срабатывании интервала пишет строку прогресса в stderr (`\r`-обновление).

**Логика / What.** Выводит обработанные строки/байты и текущую скорость не чаще `every_ms`,
чтобы не засорять stderr и не тормозить горячий путь. Активен только для интерактивного
stderr (TTY) либо при явном флаге. **Why:** наблюдаемость долгих прогонов терабайтных
файлов (→ `R6`), без влияния на пропускную способность (→ `R1`).

### Фабрики, конструкторы

Неизменяемые структуры создаются фабриками `make_*` — единый стиль конструирования и точка
проверки инвариантов (как `make_Task`/`make_Worker` в смежных проектах).

| Фабрика                  | Возвращает       | Назначение / проверки                                                        |
| ------------------------ | ---------------- | --------------------------------------------------------------------------- |
| `make_Config(...)`       | `Config`         | Проверяет непустоту `inputs`/`output_path`, `batch_rows ≥ 1`                 |
| `make_ColumnSchema(...)` | `ColumnSchema`   | Непустые `name`/`json_path`; `json_path` по умолчанию = `name`              |
| `make_InferredSchema(...)`| `InferredSchema`| Проверяет уникальность имён колонок                                          |
| `make_ParseError(...)`   | `ParseError`     | `line_no ≥ 1`, непустой `reason`, усечение `raw`                            |
| `make_ConvertStats()`    | `ConvertStats`   | Нулевые счётчики, `final_state = INIT`                                       |

## Публичный контракт

### C++ API

> Контракт функциональный: единственная точка входа — свободная функция [[#convert]].
> Вспомогательные функции из [[#Функции]] публичны для тонкого тестирования, но типовой
> потребитель вызывает только `convert`. Модель ошибок единообразна: восстановимые отказы —
> `std::expected<T, std::string>`; фатальные нарушения инвариантов конфигурации — исключение
> `std::invalid_argument` из фабрик `make_*`.

#### convert (библиотечный вход)

**Направление:** Библиотека (вызов потребителем)

**Format:** `#include <riffle.hpp>` → `ConvertStats convert(const Config& cfg);`

**Input:**

| Поле  | Тип            | Описание                                  |
| ----- | -------------- | ----------------------------------------- |
| `cfg` | `const Config&`| Конфигурация прогона (см. [[#Config]])    |

**Output:**

| Поле     | Тип            | Описание                                  |
| -------- | -------------- | ----------------------------------------- |
| `return` | `ConvertStats` | Итог прогона (см. [[#ConvertStats]])      |

**Errors:**

| Условие                                   | Поведение                                            |
| ----------------------------------------- | ---------------------------------------------------- |
| Невалидный `Config` (пустой вход и т.п.)  | `std::invalid_argument` из `make_Config`             |
| Ошибка чтения входа                       | `ConvertStats.final_state = ABORTED`; сообщение в stderr |
| Конфликт типов при `TypeConflictPolicy::ERROR` | `ABORTED`                                       |
| Ошибка записи выхода                       | `ABORTED`                                            |

**Constants:** `DEFAULT_BATCH_ROWS`, `MAX_BATCH_BYTES`, `DEFAULT_COMPRESSION`,
`INFER_SAMPLE_ROWS` — значения по умолчанию полей `Config`.

**Вызывает процессы:** [[#convert]] (и через него — весь пайплайн [[#Функции]]).

```cpp
#include <riffle.hpp>

int main() {
    riffle::Config cfg = riffle::make_Config(
        /*inputs=*/{"events.jsonl"},
        /*output_path=*/"events.parquet");
    riffle::ConvertStats stats = riffle::convert(cfg);
    return stats.final_state == riffle::PipelineState::DONE ? 0 : 1;
}
```

### CLI

Тонкая обёртка над [[#convert (библиотечный вход)]]: разбирает argv в [[#Config]], вызывает
`convert`, печатает [[#ConvertStats]] и выбирает код выхода.

#### Конвертация (CLI)

**Направление:** Пользователь → процесс

**Format:** `riffle <inputs...> -o <output> [флаги]`

**Input:** позиционные `inputs` (пути или `-`), флаги (ниже).
**Output:** выходной файл по `-o`; статистика/ошибки — в stderr.

**Вызывает процессы:** [[#convert]].

#### Флаги CLI

| Флаг                | Аргумент        | По умолчанию          | Маппинг в `Config` / процесс                          |
| ------------------- | --------------- | --------------------- | ----------------------------------------------------- |
| `-o, --output`      | путь            | — (обязателен)        | `Config.output_path`                                  |
| `--format`          | `parquet`\|`columnar-raw` | `parquet`   | `Config.output_format` → [[#open_writer]]             |
| `--schema`          | путь к JSON     | нет                   | `Config.schema_override` → [[#merge_override]]        |
| `--compression`     | `none`\|`snappy`\|`zstd` | `zstd`       | `Config.compression` → [[#open_writer]]               |
| `--batch-rows`      | целое           | `DEFAULT_BATCH_ROWS`  | `Config.batch_rows` → [[#append_row]]                 |
| `--on-error`        | `skip`\|`abort`\|`collect` | `skip`     | `Config.on_error` → [[#handle_parse_error]]           |
| `--type-conflict`   | `widen`\|`string`\|`error` | `widen`    | `Config.type_conflict` → [[#resolve_type_conflict]]   |
| `--stats`           | —               | выкл.                 | `Config.emit_stats` → [[#emit_stats]]                 |
| `-` (позиционный)   | —               | —                     | Элемент `Config.inputs` = `stdin`                     |
| `-h, --help`        | —               | —                     | Печать usage, `EXIT_OK`                               |

#### Коды выхода

| Код          | Константа       | Условие                                                  |
| ------------ | --------------- | -------------------------------------------------------- |
| `0`          | `EXIT_OK`       | `final_state == DONE`                                    |
| `2`          | `EXIT_USAGE`    | Ошибка разбора аргументов / отсутствует `-o`             |
| `3`          | `EXIT_INPUT`    | Вход не открылся / ошибка чтения                         |
| `4`          | `EXIT_DATA`     | `ABORTED` из-за данных (`OnError::ABORT` / конфликт типов)|
| `5`          | `EXIT_OUTPUT`   | Ошибка создания/записи/финализации выходного файла        |

#### Формат ошибок stderr

**Format:** `riffle: <контекст>: <причина>` — одна строка на ошибку.

| Условие                       | Сообщение (пример)                                            |
| ----------------------------- | ------------------------------------------------------------ |
| Нет `-o`                      | `riffle: usage: output path required (-o)`            |
| Вход не найден                | `riffle: input: cannot open 'events.jsonl'`          |
| Битая строка (`abort`)        | `riffle: data: line 4213: invalid JSON`              |
| Конфликт типов (`error`)      | `riffle: schema: column 'id': int64 vs string`       |
| Ошибка записи                 | `riffle: output: failed to finalize 'out.parquet'`   |

**Constants:** `SERVICE_NAME` как префикс; коды выхода — из [[#Коды выхода]].
